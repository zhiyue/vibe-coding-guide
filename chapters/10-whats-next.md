# 第十章：下一步去哪里

> **本章你将学到：** 继续成长的方向、推荐的学习资源和社区，以及一些临别建议。

## 恭喜你走到了这里

如果你从第一章读到了这里，甚至跟着做了自己的项目——那你已经完成了一件了不起的事。

你从完全不懂编程，到现在能用 AI 做出网页、小工具、甚至有后端的应用。这个变化，在几年前是不可想象的。

接下来的路，有很多方向可以走。我列出几个最值得尝试的。

## 方向 1：继续用 Vibe Coding 做更多东西

最好的学习方式就是**做更多项目**。

**挑战自己做更复杂的东西：**
- 做一个完整的多页面应用（不只是单页面）
- 试试做一个移动端适配的 Web 应用
- 挑战一个需要后端的项目（比如案例 3 那样的）
- 尝试做一个"别人也能用"的工具——分享给朋友，看看他们的反馈

**尝试更多工具：**
- 如果你一直用 Bolt.new，试试 Trae 或 Cursor
- 如果你一直用一个工具，换一个试试——不同工具的体验差异会帮你更深入地理解 vibe coding
- 关注新出的工具——这个领域变化很快，每隔几个月就有新产品出来

**参加社区活动：**
- 在 GitHub 上开源你的项目（就像你现在正在读的这个教程一样）
- 参加 vibe coding 相关的黑客马拉松或者编程挑战赛
- 在社交媒体上分享你的作品——你会发现很多人对 vibe coding 同样感兴趣

## 方向 2：学一点基础知识（可选但推荐）

你完全可以不学任何编程知识，继续用 vibe coding 做东西。但如果你愿意花一点时间了解基础，你的 vibe coding 效率会提升很多。

**为什么？** 因为当你能看懂 AI 生成的代码的大致结构时，你就能：
- 更精确地告诉 AI 哪里需要修改
- 更快地发现问题出在哪里
- 自己做一些小调整，而不需要每次都让 AI 改

**推荐从 HTML/CSS 开始。** 这是网页的"骨骼"（HTML）和"皮肤"（CSS），是最直观、最容易上手的。你不需要系统地学习一整套课程——遇到不懂的地方查一下就行。

**推荐的中文学习资源：**

- **菜鸟教程**（runoob.com）——最受欢迎的中文编程入门教程网站，有大量例子和在线编辑器，可以边学边试
- **MDN Web Docs 中文版**（developer.mozilla.org/zh-CN）——Mozilla 维护的权威 Web 技术文档，有完整的中文翻译
- **freeCodeCamp 中文社区**（chinese.freecodecamp.org）——免费的交互式编程课程，从零开始学 HTML/CSS/JavaScript
- **DataWhale Vibe-Vibe 教程**（vibevibe.cn）——国内开源社区 DataWhale 出品的系统化 vibe coding 教程，从基础到全栈

这些资源都是免费的。你不需要一口气学完，把它们当作"字典"来用——做项目的时候遇到看不懂的东西，去查一下就好。

## 本章小结

- **继续创作：** 尝试更复杂的项目、探索更多工具
- **学点基础：** HTML/CSS 是最值得了解的基础知识，能让你的 vibe coding 更高效

## 结语

一年前，"用自然语言做出一个网站"还像是科幻。现在，你不仅知道这是可能的，而且你已经亲手做到了。

Vibe coding 让编程不再是少数人的特权。它不会取代专业的软件工程师——就像自动翻译不会取代专业翻译一样。但它确实给了每一个有想法的人一种把想法变成现实的能力。

**最好的学习方式就是去做。** 不要等到"准备好了"再开始下一个项目——现在就打开工具，想一个你想做的东西，开始和 AI 对话。

不要追求完美。**先做出来，再慢慢改。** 每一个不完美的作品，都比一个完美的想法更有价值。

Vibe coding 的工具每天都在变得更好。今天你觉得有点难的事情，过几个月可能变得轻而易举。保持好奇心，保持动手的习惯，你会发现自己能做的事情越来越多。

**祝你创造出属于自己的东西。**
