# 第一章：什么是 Vibe Coding

> **本章你将学到：** 什么是 vibe coding，它从哪里来，能做什么，不能做什么。

## 从一个场景说起

假设你有一个想法——比如做一个自己的个人网站，展示你的摄影作品。

在以前，你面前只有两条路：要么花几个月学编程，自己写代码把它做出来；要么花钱请一个程序员帮你做。不管哪条路，对一个没有技术背景的普通人来说，都不轻松。

但现在，有了第三条路。

你只需要打开一个 AI 工具，用大白话告诉它："帮我做一个简洁的摄影作品集网站，左边是照片列表，点击可以放大看，整体风格用黑白灰。" 然后，你就能亲眼看着一个网站在几分钟内被"生成"出来。

这就是 **Vibe Coding**。

## Vibe Coding 的由来

"Vibe Coding"这个词，来自 AI 领域的大佬 **Andrej Karpathy**。他是 OpenAI 的联合创始人之一，也曾是特斯拉自动驾驶的 AI 负责人——总之，是一位站在 AI 技术最前沿的人。

2025 年 2 月，他在社交媒体上发了这样一段话（大意翻译）：

> "有一种新的编程方式，我称之为 'vibe coding'。你完全沉浸在感觉中，拥抱这种指数级的变化，忘记代码的存在。我在构建一个项目，但这并不算真正的编程——我只是看看结果、说说想法、跑跑程序、复制粘贴一下，大部分时候它就能工作。"

"Vibe"这个英文词的意思是"氛围""感觉"。所谓 vibe coding，就是**凭感觉写代码**——你不需要懂代码是什么，只要告诉 AI 你的感觉和想法，它来帮你把代码写出来。

这条推文获得了超过 450 万次浏览，"vibe coding"这个词迅速火遍了整个科技圈。到了 2025 年底，柯林斯英语词典甚至把它评为了**年度词汇**——可见它影响有多大。

## 用类比理解 Vibe Coding

如果上面的解释还有点抽象，这里有两个类比可以帮你理解：

**类比 1：你是老板，AI 是员工**

想象你开了一家公司，你是老板，AI 是你的全能员工。你不需要亲自干活，你只需要说清楚你想要什么——"帮我做一个客户登记表"，"首页的按钮颜色换成蓝色"，"加一个搜索功能"。AI 听了之后就去埋头干活，做完了给你看，你觉得不满意，就再提意见让它改。

**类比 2：和一个全能助手对话**

就像你和一个什么都会做的助手聊天。你说"我想要一个能记录每天心情的小工具"，它就帮你做出来。你看了一眼说"颜色太暗了，换成暖色调"，它立马改好给你看。整个过程就是你说、它做、你看、你再说——如此反复，直到你满意为止。

## Vibe Coding vs 传统编程

为了让你更直观地感受两者的区别，我做了一张对比表：

| 对比项 | 传统编程 | Vibe Coding |
|--------|----------|-------------|
| 你需要学什么 | 编程语言的语法、算法、数据结构 | 如何清楚地描述你想要什么 |
| 写代码的人 | 你自己 | AI |
| 花时间最多的事 | 写代码和调试 bug | 想清楚需求、和 AI 沟通 |
| 入门门槛 | 通常需要数月学习 | 几分钟就能开始 |
| 适合做什么 | 从简单到复杂的任何软件 | 个人项目、原型验证、小工具 |
| 出错了怎么办 | 自己找 bug、自己修 | 把问题描述给 AI，让它修 |

简单来说：传统编程是你自己动手做菜，vibe coding 是你告诉厨师你想吃什么，厨师来做。

## Vibe Coding 能做什么

你可能在想：这听起来很酷，但它真的能做出东西来吗？

答案是：能，而且已经有很多真实案例了。

**个人网站和作品集**——这是最常见的 vibe coding 项目。用几句话描述你想要的风格和内容，十几分钟就能做出一个像样的个人主页。

**小工具和小应用**——比如记账工具、打卡应用、倒计时器、待办清单。这些功能不复杂但很实用的东西，特别适合用 vibe coding 来做。

**真实的创业项目**——独立开发者 Pieter Levels 用 AI 编程工具在大约 3 小时内做出了一个 3D 多人在线飞行模拟器游戏，后来每月收入超过 5 万美元。而 Karpathy 本人也在一次编程马拉松中，用纯 AI 编码做出了一个完整的 Web 应用——拍一张菜单照片，AI 就能为每道菜生成对应的图片。

## 诚实说说局限性

说了这么多好处，我也必须诚实地说说 vibe coding 目前的局限：

**它不适合做大型团队协作项目。** 当一个项目需要十几个人一起开发、代码量达到几十万行的时候，vibe coding 的"凭感觉"模式就不够用了。大型项目需要严谨的架构设计和团队协作流程。

**它不适合对安全要求极高的场景。** 如果你要做一个涉及资金交易的金融系统，或者处理敏感医疗数据的应用，光靠 vibe coding 是不够的——这些场景需要专业的安全审查。

**AI 会犯错，需要你来验收。** AI 不是万能的，它写的代码有时候会有 bug，做出来的界面有时候和你想的不一样。你需要学会检查结果，发现问题时知道怎么让 AI 修改。（别担心，我们在第 7 章会专门教你怎么应对这些情况。）

**越复杂的项目，需要越多耐心和技巧。** 做一个简单的单页面网站可能 10 分钟就搞定了，但做一个有用户登录、数据存储、多页面的完整应用，可能需要几个小时甚至更长时间，也需要你掌握更多和 AI 沟通的技巧。

总的来说，**vibe coding 是一个非常强大的工具，但它不是魔法。** 把它想象成一个非常能干但偶尔会犯错的助手——你需要学会怎么和它配合，才能发挥出它的最大价值。

## 本章小结

- **Vibe Coding** 是一种用自然语言告诉 AI 你的需求，让 AI 帮你写代码的新方式
- 这个概念由 AI 专家 Andrej Karpathy 在 2025 年 2 月提出
- 核心是"你说想法，AI 写代码"——你不需要学编程语言
- 它适合个人项目、小工具、原型验证，但不适合大型团队项目或高安全要求场景
- AI 会犯错，你需要学会检查和反馈
